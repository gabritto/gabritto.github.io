Academic

  • Formal education

  • Research: TSDolly

    For my undergraduate thesis, I was motivated by an internship experience to try and improve the quality of TypeScript automated refactorings. Advised by Leopoldo Motta Teixeira, I researched how to test tools that modify TypeScript programs.

    The result was TSDolly, a tool that generates TypeScript programs which can be used as test inputs for testing tools. The programs TSDolly generates satisfy preconditions imposed by the tools under test: for example, to be able to test a refactoring that modifies a class, we need to make sure the generated test input program actually has a class declaration.

    TSDolly implements a technique that allows for generating programs that satisfy user-specified preconditions. This technique was adapted from previous work focused on generating Java programs suitable for testing Java automated refactorings.
    To demonstrate the use of TSDolly, I used it to test some TypeScript refactorings. Testing these refactorings by applying them to the TSDolly generated programs, I was able to find and report a bug in a TypeScript refactoring.

    My thesis can be found here (in English, abstract also in Portuguese).
    As a result of this undergraduate thesis work, my advisor Leopoldo, Rohit Gheyi and I wrote a paper to be published in the 25th Brazilian Symposium on Programming Languages (SBLP'21). A preprint can be found here (in English).

Internships

While I was a student, I worked as a Software Engineer Intern several times:

  • TypeScript Compiler, Microsoft

    December 2019 - March 2020

    I worked on improving dts-critic, a tool that checks the compatibility between a TypeScript declaration file (a declarations-only file declaring variables, functions etc, and their types) and a JavaScript library.
    I contributed additional checks for some of JavaScript's many ways to specify modules, and their corresponding declaration pattern in TypeScript.
    dts-critic is used on the DefinitelyTyped repository, an open source collection of TypeScript declarations for JavaScript libraries, written by the TypeScript community. The tool runs as a part of the tests that are executed for every contribution to DefinitelyTyped, aiming to improve the quality of contributed declarations.

  • TypeScript Compiler, Microsoft

    January 2019 - March 2019

    I developed a TypeScript refactoring for emulating the named parameters language feature (also known as named arguments or keyword arguments).
    JavaScript, and therefore TypeScript, does not provide builtin support for named parameters. A way to emulate this feature is to use an object as a function's parameter, where each of the object's properties is a named/keyword parameter. Then, the function can be called with an object argument, where each property of this argument object corresponds to a named (keyword) argument.

    An illustrative example is that this refactoring would convert the code 
    													function getName(firstName: string, lastName: string) {
		return firstName + " " + lastName;
	}
	getName("Barbara", "Liskov");
    into 
    													function getName({ firstName: string, lastName: string }) {
		return firstName + " " + lastName;
	}
	getName({ firstName: "Barbara", lastName: "Liskov" });

The component was developed using C#, CSS, and TypeScript.
